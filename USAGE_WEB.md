# Web UI 使用手册

可视化界面是用 Flask + Alpine.js 写的本地 Web，启动后浏览器打开即可——零安装、零部署。

---

## 启动

```bash
python run_web.py
# 默认 http://127.0.0.1:5000
```

可选参数：

```bash
python run_web.py --host 0.0.0.0 --port 8080   # 局域网访问 / 自定义端口
python run_web.py --debug                       # 调试模式（代码改了自动 reload）
```

启动后**保持终端不关**，浏览器打开 `http://127.0.0.1:5000`。

## 第一次使用 · 完整流程

### 1. 配置 LLM（首次必做）

启动后顶栏会有 **⚙️ 模型设置** 按钮，点开：

- 看到 `user_models.json` 里配的所有模型 profile
- 可以在 web 上新增 / 修改 / 删除 profile（API key 直接编辑）
- 每个 profile 的 `usage` 字段可勾选：
  - **main** — 写作主模型（必选一个）
  - **reviewer** — 审核员模型
  - **fallback** — 主模型失败兜底
  - 不勾选 `usage` 的 profile 仅供 `state.power_system.special_abilities[*].external_llm_profile` 显式按 id 绑定（叙事内 AI 用，如豆包）

### 2. 新建小说（两种入口）

#### a. 选择题向导（小白友好）

新手推荐。5 步选择题：
1. 你要写哪类故事？（穿越/重生/仙侠/玄幻/科幻/言情/...）
2. 故事走什么路子？（金手指逆袭/复仇/养成/...）
3. 主角是什么类型？（隐藏王者/废柴觉醒/老实人/...）
4. 整体什么基调？（热血/沉稳/搞笑/暗黑/...）
5. 给哪个平台/读者？（起点男频/晋江女频/番茄...）

每步选完后系统自动凑出 intent 文本。点 **生成意图** 进入下一步预览。

#### b. 高级模式（直接写意图）

直接粘贴一段 200-500 字的自然语言描述：

```
主角林砚是 35 岁的 AI 实验室研究员，意外车祸后穿越成大雍朝
寒门秀才沈砚。穿越时电脑上未关的"豆包"AI 网页随他意识同来到
古代，成为脑内可随时调用的智能助手。开场父亲刚去世、母亲
病重、家中欠下三十两高利贷，债主三日内来收房，主角必须靠
豆包提供的现代知识（化学/医药/商业/律法原理）在三天内筹款
救母，开启从寒门秀才到改革者的逆袭之路。题材：穿越商战 +
寒门重生 + AI 金手指。
```

意图越具体、规划越准。**避免太抽象**（"主角是个普通人"这种远没用）。

填完点 **创建项目**。

### 3. 选模式（stepwise 还是 auto）

新建后顶栏出现项目，点 **▶ 启动** 之前可选模式：

| 模式 | 行为 |
|---|---|
| **stepwise**（推荐新人） | 每个阶段组（意图 → 世界 → 人物 → 情节 → 卷/章）跑完后**暂停**，等你在左侧菜单审完产物再点继续 |
| **auto** | 一路跑通不暂停（除非遇到 HITL 关卡）。适合**对系统熟悉之后**节省手动操作 |

切换：顶栏 **模式** 切换按钮。

### 4. 启动 + 看进度

点 **▶ 启动**。后台子进程启动跑 director.py 流水线。

观察：
- **顶栏状态徽章**：当前 phase / 当前 agent
- **底栏进度**：phase 完成数 / 总数
- **左侧菜单**：当前 phase 对应的菜单项会**高亮**（"running"样式）；跑完后变成已完成
- **stdout 日志**：点击顶栏 **📃 日志** 按钮看实时输出

### 5. stepwise 暂停后审产物

跑完一个阶段组（如意图组 G1）后子进程会自动暂停。这时**左侧菜单**所有该组的面板都有数据：

| 阶段 1 立项的可看项 |
|---|
| ✨ **创作意图** — 题材 / 主角原型 / 卖点 / 对标作品 |
| **卖点定位** — ConceptPitch（吸引读者的 1-2 句话） |
| **套路库** — 拥抱 / 规避的套路清单 |
| **文风手册** — 基调 / 视角 / 慎用词 |
| **全书蓝图** — MasterOutline：22 角色槽位 / 6 层势力 / 10 关键节点 |
| **主角内核** — 核心创伤 / 真实目标 / 致命弱点 |

逐项点开看，**不满意就改** —— 大部分面板支持直接编辑保存：

- **直接改**：点字段进入编辑框，改完点保存
- **重新生成**：每个面板顶部有 **🔄 重生** 按钮，让系统重跑这个 phase（注意会消耗 token）

### 6. 进入下一阶段组

审完点顶栏 **▶ 继续**，子进程从断点继续跑下一组。

### 7. 跑到卷级 / 章节阶段

跑完 G1-G4 + 卷级规划后，**章节写作开始**。

每写完一章：
- **左侧菜单 → 章节列表** 出现新章节标题
- 点击进入**章节面板**看：
  - 正文（可滚动阅读）
  - 7 维度评分（叙事/张力/角色/钩子/结构/主角/表达 等）
  - 各类章后审计的结果（能力 / 读者体验 / 对话质量 / 蓝图遵循）
  - **✍ 重写** 按钮：填反馈后重写本章
  - **💬 对话编辑器**：用自然语言跟 AI 对话，让它按你要求改章节某段

## 几个关键操作

### A. 重写一章（带反馈）

任何已写的章节都可以重写：

1. 左侧菜单 → 章节列表 → 点要重写的章节
2. 顶部 **✍ 重写本章** 按钮
3. 弹出输入框写**作者反馈**：

   ```
   主角性格不对——本章主角太冷酷，应该多点焦虑/犹豫的内心戏。
   另外把第三幕"豆包说……"那段改成 [[ASK_AI:豆包|……]] 占位。
   ```

4. 点确认。系统：
   - 自动删旧稿
   - 清理 18 类章级派生数据（记忆 / 审计 / 关系演化等）
   - 用 feedback 重写
   - 跑全套审校 + 章后审计

### B. 章节对话编辑器（💬）

不想整章重写、只想改某段？用对话编辑器：

1. 章节面板顶部 **💬 对话编辑** 按钮
2. 像跟 ChatGPT 聊天一样描述要怎么改：

   ```
   你: 把开头 200 字的环境描写删掉，从主角推门进来开始写
   AI: 已修改。新开头你看下：……
   你: 好的，但改完后让母亲先咳嗽两声再说话
   AI: ……
   ```

3. 改满意后点 **应用到正文**

每轮对话都基于当前正文 + 上下文（直接发给主写模型），改完写回章节文件。

### C. 暂停 / 恢复

写作中可以随时暂停：

- 顶栏 **⏸ 暂停** —— 当前 phase 跑完后停下来等
- 顶栏 **▶ 继续** —— 接着跑

如果电脑断电 / 进程崩了：
- 重启 `python run_web.py`
- 顶栏选回原项目
- 点 **▶ 继续** —— 自动从最后断点恢复（progress.json 记录了已完成 phase）

### D. 版本快照 + 回滚

每个 phase 完成都会留快照。**元数据 → 版本快照** 面板能看：

- 时间线：每个快照的时间戳 + 标签 + 当时进度
- 任何一个快照都可以 **🔙 回滚** —— 把 state 回退到那一刻（章节文件不动，需自己手动删）

### E. 重生某个 phase（不影响其他 phase）

如果发现"卖点定位"产物不满意，但不想整本重头：

1. 左侧菜单 → 卖点定位面板
2. **🔄 重生本 phase** 按钮
3. 系统会：
   - 把这个 phase 标为"未完成"
   - 清掉相关产物
   - 重跑该 phase（其他 phase 数据保留）
   - 注意：**下游依赖此 phase 的产物可能需要级联重生**——系统会提示

### F. HITL 关卡（人在回路）

某些关键时刻系统会**强制暂停**等你审：

| 触发条件 | 你要做什么 |
|---|---|
| 主角境界突破 | 审核新境界对故事节奏是否合适 |
| 主线伏笔回收 | 审核回收方式是否符合前面铺垫 |
| 重要角色死亡 | 审核死亡时机 / 影响 |
| Stage 审查 critical 多 | 审核是否真的要重写多章 |

弹出审核窗口：批准 / 修改 / 拒绝。

`config.py` 里 `HITL_MODE` 可选 `auto`（不强制） / `pause`（默认） / `skip`（关闭所有 HITL）。

## 常见问题

### Q: 启动后浏览器打不开 / 一直转圈？

A: 检查
1. PowerShell 终端那里有没有报错（`python run_web.py` 启动信息正常吗？）
2. 浏览器试 `http://127.0.0.1:5000/api/projects` —— 应该返回 JSON。打不开就是后端没起
3. 如果挂了 VPN，确认 VPN 不劫持 localhost（参考 v2 项目的 STUDIO.md 配置）

### Q: 跑到一半卡死？

A: 顶栏 **🛑 强制停止** 按钮（会写 stop.flag 让子进程优雅退出），3 秒不退强 kill。如果连 web 都没响应，PowerShell 终端 Ctrl+C 重启 `python run_web.py`。下次启动会自动从断点恢复。

### Q: token 消耗想看实时？

A: 底栏左侧"LLM 池"指示器：并发数 / 当前 RPM / 熔断状态 / 累计调用次数。具体 token 数得自己去 LLM 服务商后台看（OpenAI / Anthropic 等）。

### Q: 想跟着系统改 prompt，不重新启动行不？

A: `python run_web.py --debug` 会自动 reload 改过的 py 文件。但**子进程**（director.py 主流程）改了之后**必须重启项目**才能生效。

### Q: 多个项目同时跑？

A: 一个 web 实例**只能跑一个项目**子进程；多项目要等当前 idle 后切换。如果想并行：开两个 web 实例（端口不同）。

## API 端点（给想做集成的开发者）

REST 端点全在 `web/app.py`，主要：

| 端点 | 干啥 |
|---|---|
| `GET /api/projects` | 列出所有项目 |
| `POST /api/projects` | 新建项目 |
| `POST /api/projects/<id>/start` | 启动 |
| `POST /api/projects/<id>/pause` / `resume` / `stop` | 控制 |
| `GET /api/projects/<id>/status` | 实时状态（current_phase / progress）|
| `GET /api/projects/<id>/log?lines=200` | 读 stdout 日志 |
| `GET /api/state` | 当前项目的完整 NovelState |
| `GET /api/section/<name>` | 取某个面板的数据 |
| `PUT /api/section/<name>` | 改某个面板的数据 |
| `POST /api/regen/<action>` | 重生某 phase |
| `POST /api/projects/<id>/chapter/<n>/rewrite` | 重写某章 |
| `GET /api/versions` / `POST /api/rollback/<ts>` | 快照 / 回滚 |
| `GET/POST/PUT/DELETE /api/user_models` | LLM 模型管理 |
| `GET /api/llm_pool/stats` | LLM 池统计 |

详细签名见 `web/app.py` 顶部的 `@app.route` 装饰器。
